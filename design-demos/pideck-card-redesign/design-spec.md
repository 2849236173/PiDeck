# PiDeck 卡片式前端重设计 · Design Spec

## 1. 产品概述

PiDeck 是一款面向本地开发工作的 Electron 桌面应用，用于在多个项目目录之间管理和运行 pi RPC Agent。应用提供多项目工作区、会话时间线、历史会话恢复、文件抽屉、Git 分支信息、模型选择、工具调用展示以及打包发布能力。

本次任务不是功能改造，而是**在保持所有按钮功能不变的前提下，对现有主界面做卡片式视觉重设计**。产出为 HTML 静态高保真原型，供用户后续优化 Electron 前端时参考。

## 2. 目标受众与使用场景

- 开发者、工程师、技术团队管理者
- 在本地同时维护多个项目，每个项目下运行多个 pi Agent 会话
- 高频操作：切换项目/Agent、查看会话历史、发送命令、查看终端输出、管理文件上下文
- 使用环境：桌面端，macOS/Windows，窗口宽度通常在 1280–1920px 之间

## 3. 核心界面区域（内容要点）

必须保留以下功能区域，仅改变视觉呈现方式：

1. **顶部标题栏（custom titlebar）**
   - 左侧 macOS 三色按钮 + 置顶 Pin
   - 右侧窗口控制（最小化/最大化/关闭）

2. **左侧项目/Agent 边栏**
   - 品牌区：PiDeck logo（绿色 π 图标 + PiDeck 字标）
   - 搜索框 + 新建项目按钮
   - 项目/Agent/会话层级列表
   - 底部工具按钮：设置、配置、反馈、主页、折叠边栏

3. **中间会话区域**
   - 顶部信息条：Chat agent 标题、路径、AgentId、模型选择、思考级别、上下文使用率、上传/下载、缓存、费用
   - 新会话按钮
   - 消息列表（用户气泡、助手回复、工具调用卡片）
   - 右侧快捷工具栏：编辑、终端、文件、浏览器、飞书等

4. **底部输入区**
   - 模式选择：普通模式、模型、Prompt 模板、思考级别
   - 输入框占位文案：输入消息，Enter 发送。/命令，@文件，!shell
   - 发送按钮
   - 当前会话文件路径提示

5. **底部终端抽屉**
   - 可折叠的终端输出区域

## 4. 情感基调与气质关键词

- 安静、克制、开发工具感
- 卡片式：清晰的分层、柔和的阴影、明确的边界
- 效率优先：信息密度适中，一眼定位当前项目/Agent/会话
- 不要营销页感、不要装饰性渐变、不要过度圆角
- 品牌绿（#14b814 / #238636）作为核心 accent，保持 PiDeck 识别度

## 5. 输出格式与尺寸

- 纯 HTML + CSS（可内联少量 JS 用于交互演示），单文件或少量文件
- 浏览器中可直接打开，无需构建
- 设计画布尺寸：模拟桌面应用窗口，优先 1440×900 视口
- 三版 HTML 分别保存为：
  - `design-demos/pideck-card-redesign/v1-bauhaus.html`
  - `design-demos/pideck-card-redesign/v2-linear.html`
  - `design-demos/pideck-card-redesign/v3-braun.html`

## 6. 已知约束

- 所有按钮、选项、图标的功能必须保留，不可删除或隐藏
- 保持现有的三栏/四栏布局逻辑：左栏项目、中栏会话、右栏工具、底部终端
- 中文文案保持原意，可微调排版
- 不使用外部图片资源（用 CSS 几何、内联 SVG、品牌 logo 的 CSS 实现）
- 暗色模式可适配，但默认先出浅色模式

## 7. 图片需求

本设计为工具界面，无需真实摄影/产品图。使用以下方式替代：
- 品牌 logo 用 CSS 几何实现（绿色 π 图标）
- 头像/状态用 CSS 圆点、图标字体/SVG
- 消息气泡、工具卡片、文件卡片用 CSS 形状

## 8. 三套设计逻辑

### 逻辑一 · 秒数轮盘（Bauhaus Geometric）

从网页风格库抽取第 10 号风格：包豪斯几何标志 + 扁平插画系统。

- 视觉 DNA：三原色谱系（红/黄/蓝）+ 黑白，纯色块拼接，几何无衬线圆润字体，圆/三角/方基本几何单元搭建插画，对齐栅格、模块化拼图。
- 适配到 PiDeck：用包豪斯几何块重新组织左侧项目卡片、中间会话卡片、右侧工具卡片。品牌绿作为 accent，辅以少量原色块做状态标识。
- 还原度：90%，纯 CSS 几何可完全还原。
- 字体：Poppins / Manrope（几何圆润）+ 系统中文栈。

### 逻辑二 · 现实参照（Linear / Bento Glass）

参照 Linear 和 Cursor 的开发者工具美学：暗色底、Bento 网格、玻璃拟态、发光边框、发丝分割线。

- 视觉 DNA：近黑底 #08090A + 去饱和蓝紫品牌 #5E6AD2 + 低饱和青紫微光渐变，几何无衬线负字距紧凑，Bento 网格分块，磨砂玻璃卡片，流光 streamer。
- 适配到 PiDeck：将左中右区域处理为 Bento 卡片，会话消息卡片化，工具栏图标卡片化，输入区悬浮卡片。
- 还原度：85%，玻璃拟态与发光用 CSS 滤镜/渐变实现。
- 字体：Inter / Geist + Geist Mono。

### 逻辑三 · 最佳设计师（Dieter Rams / Braun 功能主义）

以 Dieter Rams 的「少即是多」和 Braun 工业设计为灵感：极度克制、功能优先、清晰网格、柔和灰阶、精致排版。

- 视觉 DNA：暖白/浅灰底、系统灰阶、单一品牌绿 accent、锐利但小圆角、高密度信息、卡片之间用极细边界和微妙阴影区分。
- 适配到 PiDeck：每个信息单元都是一张功能卡片，层级通过灰阶和间距而非装饰色表达。左侧项目树卡片化，中间会话卡片化，右侧工具面板卡片化，底部终端嵌入式卡片。
- 还原度：98%，纯 CSS 可完全还原。
- 字体：系统无衬线 + 等宽字（PiDeckCommitMono）用于技术数据。

## 9. 自检清单

- [ ] 三版 HTML 都保留完整的功能区域
- [ ] 三版都是卡片式视觉，但气质差异明显
- [ ] 没有使用装饰性紫色渐变、emoji 图标、烂大街圆角卡片+左 border accent
- [ ] 品牌绿被保留
- [ ] 每版都有 hover/active/focus-visible 状态
- [ ] 在浏览器中可正常打开，无控制台报错
